# Welcome to MyJpa-Plus

MyJpa-Plus is a type-safe JPA Specification builder with a fluent Lambda API for Spring Data JPA.

## Why MyJpa-Plus?

Traditional JPA Criteria API is verbose and error-prone:

```java
// Traditional way - error-prone
Root<User> root = query.from(User.class);
Predicate predicate = cb.and(
    cb.equal(root.get("status"), "ACTIVE"),      // Magic strings!
    cb.like(root.get("name"), "%John%"),          // Typo risk!
    cb.greaterThan(root.get("age"), 18)
);
```

MyJpa-Plus provides a clean, type-safe alternative:

```java
// MyJpa-Plus - Type-safe
QuerySpec<User> spec = new QuerySpec<User>()
    .eq(User::getStatus, "ACTIVE")      // Compile-time check!
    .like(User::getName, "John")        // No magic strings! Auto wraps with %
    .gt(User::getAge, 18);

// Pass directly to findAll()
userRepository.findAll(spec);
```

## Key Advantages

| Feature | Traditional JPA | MyJpa-Plus |
|---------|---------------|------------|
| Field reference | Strings (`"name"`) | Method references (`User::getName`) |
| Type safety | Runtime errors | Compile-time errors |
| Null handling | Manual checks | Auto IS NULL |
| OR/NOT groups | Complex nesting | Clean Consumer API |
| JOIN conditions | Verbose code | Fluent builder |

## Quick Start

### Add Dependency

```xml
<dependency>
    <groupId>io.github.zsubera</groupId>
    <artifactId>myjpa-plus</artifactId>
    <version>1.3.1</version>
</dependency>
```

### Basic Usage

```java
// Define Repository
public interface UserRepository extends MyJpaRepository<User, Long> {
}

// Build queries with QuerySpec
QuerySpec<User> spec = QuerySpec.of(s -> s
    .eq(User::getStatus, "ACTIVE")
    .like(User::getName, "John")
    .gt(User::getAge, 18)
    .orderByDesc(User::getCreatedAt)
);

List<User> users = userRepository.findAll(spec);
```
