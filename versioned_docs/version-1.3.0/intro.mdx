# 欢迎使用 MyJpa-Plus

MyJpa-Plus 是一个类型安全的 JPA Specification 构建器，为 Spring Data JPA 提供流畅的 Lambda API。

## 为什么选择 MyJpa-Plus？

传统的 JPA Criteria API 冗长且容易出错：

```java
// 传统方式 - 容易出错
Root<User> root = query.from(User.class);
Predicate predicate = cb.and(
    cb.equal(root.get("status"), "ACTIVE"),      // 魔法字符串！
    cb.like(root.get("name"), "%John%"),          // 拼写错误风险！
    cb.greaterThan(root.get("age"), 18)
);
```

MyJpa-Plus 提供了简洁、类型安全的替代方案：

```java
// MyJpa-Plus - 类型安全
QuerySpec<User> spec = new QuerySpec<User>()
    .eq(User::getStatus, "ACTIVE")      // 编译时检查！
    .like(User::getName, "John")        // 没有魔法字符串！自动添加 %
    .gt(User::getAge, 18);

// 直接传递给 findAll()
userRepository.findAll(spec);
```

## 核心优势

| 特性 | 传统 JPA | MyJpa-Plus |
|------|----------|------------|
| 字段引用 | 字符串 (`"name"`) | 方法引用 (`User::getName`) |
| 类型安全 | 运行时错误 | 编译时错误 |
| 空值处理 | 手动检查 | 自动 IS NULL |
| OR/NOT 组 | 复杂嵌套 | 清晰的 Consumer API |
| JOIN 条件 | 冗长代码 | 流畅的构建器 |

## 快速开始

### 添加依赖

```xml
<dependency>
    <groupId>com.github.zsubera</groupId>
    <artifactId>myjpa-plus</artifactId>
    <version>1.3.0</version>
</dependency>
```

### 基本用法

```java
// 定义 Repository
public interface UserRepository extends MyJpaRepository<User, Long> {
}

// 使用 QuerySpec 构建查询
QuerySpec<User> spec = QuerySpec.of(s -> s
    .eq(User::getStatus, "ACTIVE")
    .like(User::getName, "John")
    .gt(User::getAge, 18)
    .orderByDesc(User::getCreatedAt)
);

List<User> users = userRepository.findAll(spec);
```
